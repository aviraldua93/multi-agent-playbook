The orchestrator accumulates everything. Quality degrades.

This Isn't Just a Company Problem
🔍
PR Reviews
Review code + analyze CI logs + check coverage = context overflow
🔄
Refactoring
One agent can't scope + implement + test across 50 files
🐛
Debugging
Parse logs + test hypotheses in parallel = needs multiple agents

From Claude Code issue #4182: "Without nested tasks, the PR review agent must process entire CI logs in its own context — thousands of lines. This leads to context rot."

How I Adapted
Docs as Message Bus

Agents can't remember. Files can.

🧠

↓↓
Agent A
→
📄 findings.md
→
Agent B

// Agent B reads what Agent A wrote.
// The repo is the shared memory. ✓
